Roomzin Documentation

Reference guide for installation, configuration, APIs, and SDKs.

1. Installation

Download the latest release from GitHub and run the binary:

curl -L https://github.com/roomzin/releases/latest/download/roomzin.tar.gz | tar xz
cd roomzin
./roomzin --help

Supported platforms: Linux, macOS. Windows builds available on request.

2. Configuration

Roomzin can be configured via a YAML file or environment variables.

server:
  id: node-1
  port: 8080
  data_dir: /var/lib/roomzin

cluster:
  peers:
    - http://node-1:8080
    - http://node-2:8080
    - http://node-3:8080

security:
  enable_tls: true
  cert_file: /etc/roomzin/cert.pem
  key_file: /etc/roomzin/key.pem

Minimal configuration requires only a server.id and port.

3. Cluster Setup

Roomzin supports both standalone and clustered deployments.

  • Standalone: start a single node with ./roomzin --standalone.
  • Clustered: launch multiple nodes with shared peer addresses. The cluster elects a leader automatically.

Clusters use a Raft-like consensus mechanism with snapshots and WAL for consistency.

4. Admin API

The Admin API is available on every node for health checks, metrics, and cluster management.

GET /admin/health     → {"status":"ok"}
GET /admin/metrics    → Prometheus-compatible metrics
GET /admin/peers      → List cluster peers
POST /admin/snapshot  → Trigger snapshot save

All endpoints require authentication if security is enabled.

5. SDKs Overview

Official SDKs handle cluster routing, retries, and failover automatically:

  • Java SDK: rz-java
  • Go SDK: rz-go
  • Python SDK: rz-py
  • .NET SDK: rz-dotnet

Each SDK provides:

  • Cluster discovery
  • Automatic retries
  • High-level query API

See Tutorials for integration examples in each language.
