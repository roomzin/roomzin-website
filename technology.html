Technology Architecture

Built in Rust. Optimized for inventory. Proven at scale.

Architecture Overview

Roomzin is not a cache, and not a database — it's a partitioned in-memory inventory engine built for one job: serving travel availability and prices with ultra-low latency.

Partitioned State Machine

Each shard handles properties independently, scaling horizontally across cores and nodes

Snapshots + WAL

Fast state recovery and durability with compact snapshots and minimal write-ahead logging

Smart SDKs

Client SDKs handle routing, retries, and peer updates automatically

QUIC + TLS Security

Lightweight, secure transport optimized for low latency at the edge

Developer Experience

Developers don't need to think about nodes, peers, or routing. The SDK takes care of it automatically:

  • Provide a single endpoint, and the SDK keeps itself updated with cluster topology
  • Predefined models ensure clean, predictable API interactions
  • REST API available for management, health checks, metrics, and node info

Integration time is measured in hours, not weeks.
